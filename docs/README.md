# Sertor documentation

This is the map of Sertor's user documentation — **where to go for what**. New here? Start with
**[getting-started.md](getting-started.md)**: the single path from an unconfigured repository to your
first working retrieval. Everything else below is a branch off that path.

Everything here is **host-agnostic** — it applies whether your assistant is **Claude Code** or
**GitHub Copilot** — except the two per-assistant quick starts, which are called out as such.

## Start here

**→ [getting-started.md](getting-started.md)** — the one path from nothing to first value:
prerequisites → install → index → a first query that shows the code+doc fusion. If you read one page,
read this one.
