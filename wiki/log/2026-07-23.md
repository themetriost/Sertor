## [2026-07-23] record | E10-FEAT-040 wiki-guard — gate Stop bloccante per record + lint semantico

Implementata **E10-FEAT-040 `wiki-guard`**: hook host-facing che dà a **record + lint semantico** la
stessa rete hard che [[daily-distill-floor]] dà al distill, ma **allo Stop** (fine turno) invece che al
merge. Nasce dalla richiesta utente («le istruzioni del CLAUDE.md vengono ignorate, voglio enforcement
deterministico») dopo che il nudge non-bloccante `wiki-pending-check` veniva regolarmente saltato.

**Design (confermato leggendo i doc ufficiali, non inferendo):** letti [Claude Code hooks](https://code.claude.com/docs/en/hooks)
e [GitHub Copilot hooks reference](https://docs.github.com/en/copilot/reference/hooks-reference).
Verificato: Claude `Stop` + `{"decision":"block","reason"}` (JSON parsato solo con exit 0) **e** Copilot
`agentStop` + `{decision:block,reason}` → **parità piena** (l'utente aveva ragione: Copilot ha lo stesso
blocco allo Stop). **Scartato** l'handler `agent`/`prompt` per un verify semantico *deterministico*:
Claude-only (Copilot ha `prompt` solo su `sessionStart`, auto-submit) **e** sposterebbe il **giudizio**
dentro l'hook, contro il confine D↔N (Principio XI). Scelta: **presence-gate `command` + reason ricco** —
l'hook esige, il main agent giudica.

**Meccanismo:** rileva **riusando** `sertor-wiki-tools scan` (`wiki.scan/1`, `pending`) — niente
reinvenzione di git-status/timestamp; se `pending>0` allo Stop → blocco con reason che esige record +
distill + **lint semantico**. `stop_hook_active`-guard, fail-open (no config/scan → non intrappola,
+breadcrumb), `pending==0` (read-only) chiude normale, exit 0 sempre, stdlib-only. Sostituisce il wiring
**Stop** di `wiki-pending-check` (che resta su SessionEnd).

**Consegna:** script `wiki-guard.py` (portabile via `_hooklib`), wiring Claude `settings.hooks.json` +
Copilot `_copilot_wiki_hook_specs` in `install_wiki.py`, 8 test, sync bundle. **Gate verde** (root 1244 ·
sertor 564 · ruff clean). **Prova LIVE** contro lo script reale: sessione con lavoro non registrato →
`{decision:block}` con «2 files newer than the last log entry»; `stop_hook_active=true` → nessun output
(anti-loop). **Debito tracciato FEAT-041:** l'`upgrade` sugli ospiti esistenti deve rimuovere il vecchio
wiring Stop (gemello FEAT-031→032); fresh install e dogfood sono corretti. Entità distillata:
[[wiki-guard]]. **explainer:** non serve (tooling di governance interno). Implementazione delegata a un
fork con verifica in flusso principale.
